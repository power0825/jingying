# 审批流修改总结

**日期：** 2026-04-07

## 修改概述

根据用户需求，重构了系统中的三大审批流：立项审批、报销审批、结算单审批，实现了多级审批和基于角色的审批权限控制。

---

## 一、立项审批流

**状态定义：** `草稿` → `待初审` → `待终审` → `已通过` / `已驳回`

**审批流程：**
| 提交人 | 流程 |
|--------|------|
| 客户经理 | 提交 → 待初审 → 总监初审 → 待终审 → CEO 终审 → 已通过 |
| 客户总监/运营总监 | 提交 → 待终审（初审自动通过）→ CEO 终审 → 已通过 |

**修改文件：**
- `src/components/ProjectForm.tsx` - 提交时根据项目经理角色确定初始状态

---

## 二、报销审批流（重点修改）

**新状态定义：** `草稿` → `待总监初审` → `待 CEO 终审` → `待财务审核` → `待打款` → `已打款`

**审批流程：**
| 提交人 | 初始状态 | 后续流程 |
|--------|----------|----------|
| 客户经理 | 待总监初审 | 总监初审 → 待 CEO 终审 → CEO 终审 → 待财务审核 → 财务审核 → 待打款 → 财务打款 |
| 客户总监/运营总监 | 待 CEO 终审 | CEO 终审 → 待财务审核 → 财务审核 → 待打款 → 财务打款 |
| CEO | 待财务审核 | 财务审核 → 待打款 → 财务打款 |
| 财务 | 待 CEO 终审 | CEO 终审 → 待财务审核 → 财务审核（自动通过）→ 待打款 |

**修改文件：**
1. `src/components/finance/GeneralReimbursements.tsx` - 我的报销页面，提交逻辑
2. `src/components/finance/ReimbursementApproval.tsx` - 报销处理页面，多级审批 UI
3. `src/components/project/Financials.tsx` - 项目财务中的报销模块
4. `src/pages/ProjectProfitDetail.tsx` - 项目利润详情页，状态显示
5. `src/pages/Finance.tsx` - 财务管理页，待办计数查询
6. `src/pages/Dashboard.tsx` - 仪表盘，待我审批计数

**权限控制：**
- 待总监初审：客户总监/运营总监可审批
- 待 CEO 终审：CEO 可审批
- 待财务审核：财务可审批
- 待打款：财务可打款

---

## 三、结算单审批流

**新状态定义：** `待提交` → `待 CEO 审核` → `待财务审核` → `待付款` → `已付款` / `已驳回`

**审批流程：**
| 操作人 | 操作 | 状态变化 |
|--------|------|----------|
| 运营总监 | 提交结算单 | 待提交 → 待 CEO 审核 |
| CEO | 审核通过 | 待 CEO 审核 → 待财务审核 |
| CEO | 驳回 | 待 CEO 审核 → 已驳回 |
| 财务 | 审核通过 | 待财务审核 → 待付款 |
| 财务 | 驳回 | 待财务审核 → 已驳回 |
| 财务 | 确认付款 | 待付款 → 已付款 |

**修改文件：**
1. `src/components/finance/SettlementProcessing.tsx` - 结算单处理页面
2. `src/pages/SupplierDetails.tsx` - 供应商详情页，结算单管理

**权限控制：**
- 运营总监：提交结算单、上传发票
- CEO：审核结算单（待 CEO 审核状态）
- 财务：审核结算单（待财务审核状态）、确认付款（待付款状态）、上传付款凭证

## 五、数据库字段变更

### project_reimbursements 表
新增字段：
- `submission_date` - 提交日期
- `submitter_role` - 提交人角色（用于追踪审批流程起点）

状态字段值变更：
```sql
-- 旧值
'草稿', '待审核', '待打款', '已打款'

-- 新值
'草稿', '待总监初审', '待 CEO 终审', '待财务审核', '待打款', '已打款'
```

### supplier_settlements 表
新增字段：
- `ceo_audited_by` - CEO 审核人 ID
- `ceo_audited_at` - CEO 审核时间
- `finance_audited_by` - 财务审核人 ID
- `finance_audited_at` - 财务审核时间

状态字段值变更：
```sql
-- 旧值
'待提交', '待审核', '已审核', '已付款', '已驳回'

-- 新值
'待提交', '待 CEO 审核', '待财务审核', '待付款', '已付款', '已驳回'
```

2. 立项审批的 `initial_approval_status` 和 `final_approval_status` 字段使用独立的状态值（`待审核`/`通过`/`驳回`），与报销审批流不同
