## 六、阿里云服务选型建议

### 6.1 推荐方案一：OSS + CDN（静态托管）

**适用场景：** 纯前端静态部署，API使用现有Supabase

| 服务 | 规格 | 说明 |
|-----|------|------|
| **OSS对象存储** | 标准型 | 托管前端静态文件 |
| **CDN** | 按流量计费 | 加速静态资源访问 |
| **域名** | 已备案域名 | 需要ICP备案 |

**优点：** 成本低、部署简单、无需维护服务器

### 6.2 推荐方案二：ECS + Nginx（完整控制）

**适用场景：** 需要自定义API代理、完整控制服务器

| 服务 | 规格 | 说明 |
|-----|------|------|
| **ECS云服务器** | 1核2G起步 | 托管前端+API代理 |
| **SLB负载均衡** | 可选 | 高可用场景 |
| **RDS PostgreSQL** | 可选 | 替代Supabase（国内访问更快） |
| **OSS** | 标准型 | 文件存储（替代Supabase Storage） |
| **CDN** | 按流量计费 | 加速访问 |

### 6.3 推荐方案三：函数计算 FC（Serverless）

**适用场景：** 弹性伸缩、按需付费

| 服务 | 规格 | 说明 |
|-----|------|------|
| **函数计算 FC** | 按调用计费 | 托管前端+API函数 |
| **API网关** | 按调用计费 | API路由管理 |
| **OSS** | 标准型 | 静态文件存储 |

## 八、迁移注意事项

### 8.1 从Vercel迁移

1. **API代理迁移：** 当前 `/api/qwen` 使用Vercel Serverless Functions，需迁移至：
   - 阿里云函数计算 FC
   - 或ECS上的Node.js服务
   - 或直接前端调用（需配置CORS）

2. **环境变量：** 需在阿里云服务中重新配置

3. **构建配置：** `vercel.json` 中的重写规则需在Nginx中配置

### 8.2 Nginx配置参考

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # 前端静态文件
    root /var/www/dist;
    index index.html;
    
    # SPA路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API代理（如需要）
    location /api/ {
        proxy_pass http://localhost:3000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
